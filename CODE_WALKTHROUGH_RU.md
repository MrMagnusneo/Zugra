# Подробный разбор проекта Zugra

Этот документ объясняет код в `/home/x13/VScodeProjects/my_projects/Zugra`.
Проект сейчас состоит из консольного шахматного движка в папке `minimax` и пустых заготовок в папке `ml`.

## Карта проекта

- `minimax/board.py` - преобразование ходов/матриц/PGN, получение финальной позиции, UTF-8-отрисовка доски.
- `minimax/engine.py` - простой шахматный оценщик и движок поиска хода через negamax с alpha-beta отсечением.
- `minimax/main.py` - консольная игра: аргументы командной строки, ввод цвета/ходов, печать доски и результата.
- `ml/board.py`, `ml/model.py`, `ml/train.py` - пустые файлы, пока не содержат кода.
- `minimax/__pycache__/*.pyc` - автоматически созданный Python-байткод. Его обычно не редактируют и не объясняют как исходный код.

## Главные библиотеки и понятия

- `python-chess`, импортируется как `chess`, - внешняя библиотека для шахмат: доска `chess.Board`, ходы `chess.Move`, фигуры `chess.Piece`, PGN `chess.pgn`, проверка легальности ходов, матов, ничьих, FEN, SAN/UCI.
- `chess.Board()` создает стандартную начальную позицию.
- `chess.Board(fen)` создает позицию из FEN-строки.
- `chess.Board(None)` создает пустую доску без фигур.
- `board.legal_moves` - генератор легальных ходов текущей позиции.
- `board.push(move)` делает ход на доске.
- `board.pop()` отменяет последний ход.
- `board.parse_san("Nf3")` парсит шахматную SAN-запись.
- `chess.Move.from_uci("g1f3")` парсит UCI-запись.
- `board.outcome(claim_draw=True)` возвращает результат партии с учетом заявляемых ничьих.
- `board.result(claim_draw=True)` возвращает строку результата: например `1-0`, `0-1`, `1/2-1/2`, `*`.
- `dataclasses.dataclass` автоматически создает `__init__`, `__repr__` и другие методы для классов с полями.
- `field(default_factory=...)` создает значение по умолчанию заново для каждого объекта, а не делит один mutable-объект между всеми.
- `ABC` и `@abstractmethod` задают абстрактный базовый класс: наследник обязан реализовать методы.
- `argparse` разбирает аргументы командной строки.
- `Path` из `pathlib` работает с путями файловой системы.
- `StringIO` превращает строку в файловый объект, чтобы `chess.pgn.read_game` мог читать PGN из строки.

## minimax/board.py

### Общая роль файла

Файл отвечает не за шахматный поиск, а за "ввод-вывод" шахматных данных:
массив ходов -> PGN, PGN -> массив ходов, матрица 8x8 -> позиция, PGN -> финальная доска, доска -> текстовая UTF-8-картинка.

## Как все работает вместе

1. Пользователь запускает `minimax/main.py`.
2. `parse_args()` читает параметры `--depth`, `--fen`, `--pgn`, `--engine-color`.
3. `load_board()` создает `chess.Board`.
4. `run_game()` создает `UTF8BoardRenderer` и `ChessEngine`.
5. Пока партия не окончена, программа печатает доску.
6. Если ход компьютера, `ChessEngine.find_best_move()` перебирает ходы через negamax.
7. Внутри поиска `SimpleEvaluator.evaluate()` оценивает позиции на листьях дерева.
8. Если ход игрока, `ask_player_move()` принимает SAN или UCI, проверяет легальность и применяет ход.
9. В конце `print_game_result()` выводит результат.

## Как движок выбирает ход

Движок не "понимает" шахматы как нейросеть. Он перебирает дерево ходов до заданной глубины:

- На корне берет все легальные ходы.
- Для каждого делает ход на доске.
- Рекурсивно смотрит ответы соперника.
- Поскольку после каждого хода меняется сторона, используется negamax: оценка дочерней позиции берется с минусом.
- Если позиция закончилась матом/ничьей, возвращается терминальный счет.
- Если глубина закончилась, вызывается статическая оценка.
- Alpha-beta отсечение прекращает разбор веток, которые уже не могут улучшить найденный результат.

## Что важно помнить

- Оценка `SimpleEvaluator` очень простая: материал, центр, мобильность, пара слонов, шах.
- Здесь нет дебютной книги, таблиц окончаний, quiescence search, transposition table, iterative deepening, time management.
- Поэтому движок может делать разумные короткие тактические ходы, но на большой силе играть не будет.
- `python-chess` отвечает за правила и легальность ходов, а код проекта отвечает за выбор хода и интерфейс.
